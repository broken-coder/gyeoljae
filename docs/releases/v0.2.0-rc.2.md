# gyeoljae v0.2.0-rc.2 — Developer Preview

gyeoljae remains a **Developer Preview**. Pin the exact version and deploy
shadow-first. Interfaces, CLI flags, and state formats may change between
preview releases. Publish this RC with `npm publish --tag next` only after a
separate publish approval; preparing or merging the release PR does not
authorize publishing or other dist-tag changes.

For the approval-integrity, durable inbox/outbox, pagination, packaging, and
upgrade changes introduced by this release line, see the
[`v0.2.0-rc.1` notes](v0.2.0-rc.1.md).

## Changes since `v0.2.0-rc.1`

1. **Public technical identifiers no longer trip the sanitizer.** The
   working-tree and packed-payload gates share a narrow allowlist for common
   public prefixes such as `SHA-256`, `UTF-8`, `ISO-8601`, `HTTP-2`, `TLS-13`,
   `RFC-9110`, and `GPT-4`.
2. **Private tracker detection stays fail-closed.** Arbitrary tracker-style
   identifiers remain blocked, and near-prefix forms that merely extend an
   allowed prefix are covered by regression tests.
3. **Release metadata is synchronized.** `package.json`, `package-lock.json`,
   the installation examples, and the current package-contract references
   now identify `0.2.0-rc.2` consistently. CLI bin targets use npm's
   canonical package-relative form, and the package smoke rejects paths that
   would trigger publish-time manifest correction.

No approval validator, orchestrator, notifier, credential, or runtime adapter
logic changes in this RC refresh.

## Known limitations

- Notification delivery is **deduplicated at-least-once**; use the opt-in
  `Outbox` to reconcile the remote-post crash window.
- Local JSON state requires **exactly one writer per path** and is not written
  atomically ([single-writer contract](../deployment/local-json-state.md)).
- `gyeoljae-watch` is chat-shadow but **ledger-live** and can perform GitHub
  label, comment, and close transitions whenever its token permits.
- GitHub reads have no rate-limit retry/backoff; a rejected page fails the
  pass, which restarts from the beginning next interval.
- The package is ESM-only; CommonJS `require()` is not part of the contract.
