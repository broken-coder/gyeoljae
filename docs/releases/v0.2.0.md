# gyeoljae v0.2.0

`v0.2.0` is the first non-prerelease npm release in the `0.2` line. It
promotes the package contract and runtime behavior validated in
[`v0.2.0-rc.2`](v0.2.0-rc.2.md); it does not introduce runtime, approval-core,
credential, or adapter behavior changes beyond that release candidate.

The project remains early-stage. A non-prerelease version does not make live
ledger writes safe by default: pin deployments, start shadow-first, use a
dedicated test ledger before enabling `gyeoljae-watch`, and review every
credential-bearing deployment separately.

## Upgrade from `v0.2.0-rc.2`

- The public exports, three CLI entrypoints, Node `>=22` requirement, and
  package contents are unchanged.
- Update exact-version installation references from `gyeoljae@0.2.0-rc.2` to
  `gyeoljae@0.2.0`.
- No local-state migration is required. Preserve the one-writer-per-path
  contract for local JSON files.

## Operational limitations

- Notification delivery is **deduplicated at-least-once**. Configure the
  opt-in `Outbox` when the remote-post crash window must be reconciled.
- Local JSON state is not atomic and requires **exactly one writer per path**
  ([single-writer contract](../deployment/local-json-state.md)).
- `gyeoljae-watch` shadows chat output only; it performs live GitHub
  label/comment/close transitions whenever its credential permits.
- GitHub reads have no rate-limit retry/backoff. A rejected page fails the
  pass, which restarts from the beginning on the next interval.
- The package is ESM-only; CommonJS `require()` is not part of the contract.
